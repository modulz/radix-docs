### Installation

Install the component from your command line.

```
npm install @radix-ui/react-slider
```

Then import the component in your component file.

```
// Slider.tsx
import { Slider } from '@radix-ui/react-slider';
```

### Basic example

Create your styled slider components, then pass those styled components into `Slider` via the `as` prop:

```jsx live manual
// outer container styles
const StyledSlider = styled('div', {
  position: 'relative',
  display: 'flex',
  alignItems: 'center',
  flexShrink: 0,
  userSelect: 'none',
  touchAction: 'none',
});

// track styles
const StyledSliderTrack = styled('div', {
  backgroundColor: 'gainsboro',
  position: 'relative',
  flexGrow: 1,
});

// range styles
const StyledSliderRange = styled('div', {
  position: 'absolute',
});

// thumb styles
const StyledSliderThumb = styled('div', {
  display: 'block',
});

render(
  <Slider defaultValue={10} as={StyledSlider}>
    <Slider.Track as={StyledSliderTrack}>
      <Slider.Range as={StyledSliderRange} />
    </Slider.Track>
    <Slider.Thumb as={StyledSliderThumb} />
  </Slider>
);
```

#### Controlled state

The component will work as expected without any additional props or configuration, but if you need access to its state you can control it with the value and onChange props.

```jsx live manual
function ControlledSlider() {
  const [value, setValue] = React.useState(10);
  return (
    <Slider value={value} onChange={(newValue) => setValue(newValue)}>
      <Slider.Track>
        <Slider.Range />
      </Slider.Track>
      <Slider.Thumb />
    </Slider>
  );
}

render(<ControlledSlider />);
```

### Accessibility

- Regardless of the visible size of the track or thumb, their touch targets should be sufficiently large. The touch target can be increased by adding an invisible pseudo element to the thumb and/or track.
- The user can touch or click anywhere on the `Slider` track to activate the thumb and update the value.
- The element with the accessible role of `slider` is `SliderThumb`, not `Slider` itself. This is because `Slider` can have multiple thumbs that represent two independent values.
- The element rendered by `SliderThumb` sets a value for `aria-label` by default that represents the slider's range in a human readable format. This can be overridden by passing `aria-label` directly to `SliderThumb`.
- The element rendered by `SliderThumb` sets a value for `aria-valuemin` equal to the min prop passed to Slider.
- The element rendered by `SliderThumb` sets a value for `aria-valuemax` equal to the max prop passed to Slider.
- The element rendered by `SliderThumb` sets a value for `aria-valuenow` equal to the component's current value.
- The element rendered by `SliderThumb` sets a value for `aria-orientation` equal to the orientation prop passed to Slider.
- The `SliderThumb` must be in the document's tab sequence and has a `tabIndex` of `0` by default.
