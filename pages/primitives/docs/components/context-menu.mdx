---
title: Context Menu
name: context-menu
version: 0.0.1-1
description: Displays a menu located at the pointer, triggered by a right-click.
aria: https://www.w3.org/TR/wai-aria-practices-1.2/#menu
features:
  - Supports items, labels, groups of items.
  - Supports checkable items (single or multiple).
  - Customise side, alignment, offsets, collision handling.
  - Focus is fully managed.
  - Full keyboard navigation.
  - Typeahead support.
  - Dismissing and layering behavior is highly customisable.

## Installation

Install the component from your command line.

```bash
npm install @radix-ui/react-context-menu
```

## Anatomy

Import the components and piece the parts together.

```jsx
import {
  ContextMenu,
  Trigger,
  Content,
  Label,
  Item,
  Group,
  CheckboxItem,
  RadioGroup,
  RadioItem,
  ItemIndicator,
  Separator,
} from '@radix-ui/react-context-menu';

export default () => (
  <ContextMenu>
    <Trigger />
    <Content>
      <Trigger />
      <Content>
        <Label />
        <Item />
        <Group>
          <Item />
        </Group>
        <CheckboxItem />
        <RadioGroup>
          <RadioItem />
        </RadioGroup>
        <Item>
          <ItemIndicator />
        </Item>
        <Separator />
      </Content>
    </Content>
  </ContextMenu>
);
```

## Basic example

Create your styled context menu component from the primitive parts.

**Note:** This example uses [Stitches](https://stitches.dev) but any styling solution is [compatible](../overview/styling).

```jsx live manual
// import { styled } from 'path-to/stitches.config';
import * as ContextMenuPrimitive from '@radix-ui/react-context-menu';

const Content = styled(ContextMenuPrimitive.Content, {
  minWidth: 130,
  backgroundColor: 'white',
  borderRadius: 6,
  padding: 5,
  boxShadow: '0px 5px 15px -5px hsla(206,22%,7%,.15)',
});

const Item = styled(ContextMenuPrimitive.Item, {
  fontSize: 13,
  padding: '5px 10px',
  borderRadius: 3,
  cursor: 'default',

  '&:focus': {
    outline: 'none',
    backgroundColor: 'dodgerblue',
    color: 'white',
  },
});

export default () => (
  <ContextMenuPrimitive.Root>
    <ContextMenuPrimitive.Trigger>
      <div
        style={{
          backgroundColor: 'gainsboro',
          padding: 100,
          textAlign: 'center',
        }}
      >
        Right click anywhere
      </div>
    </ContextMenuPrimitive.Trigger>
    <Content>
      <Item>Undo</Item>
      <Item>Redo</Item>
      <Item>Cut</Item>
      <Item>Copy</Item>
      <Item>Paste</Item>
    </Content>
  </ContextMenuPrimitive.Root>
);
```

## API Reference

Adheres to the [Menu WAI-ARIA design pattern](https://www.w3.org/TR/wai-aria-practices-1.2/#menu) and uses [roving tabindex](https://www.w3.org/TR/wai-aria-practices-1.2/examples/menu-button/menu-button-actions.html) to manage focus movement among menu items.

### ContextMenu

`ContextMenu` is the root component.

### Trigger

`ContextMenuTrigger` is the area that triggers the `ContextMenu`. Wrap this around the thing you want your `ContextMenu` to open when right-clicking (or using the relevant keyboard shortcuts).

### Content

`ContextMenuContent` is the component that pops out in an open `ContextMenu`.

### Group

Used to group multiple `ContextItem`.

### Label

Uused to render a label. It won't be focusable using arrow keys.

Renders when the parent `ContextMenuCheckboxItem` or `ContextMenuRadioItem` is checked. You can style this element directly, or you can use it as a wrapper to put an icon into, or both.
